# Methodology

## Dokumentzweck

Dieser Bereich enthält die vollständige Architektur der eigenen Methodik „Generative Engine Optimization (GEO)“ von Anna Trocka.

Die Dokumente beschreiben die grundlegenden Ebenen der Methodik sowie deren maschinenlesbare Spiegelung.

---

## Struktur der Methodik

### Layer 1
GEO-Fundament

Aufbau einer stabilen Informationsarchitektur.

---

### Layer 2
Identity Confirmation

Bestätigung von Rollen, Entitäten und Vertrauensstrukturen.

---

### Layer 3
External Signals and Patterns

Verstärkung und Synchronisation von Informationsbeziehungen durch externe Signale und wiederkehrende Muster.

---

### Layer 4
Recommendation Triggers and Matching

Definition situationsbezogener Empfehlungs- und Matching-Prozesse.

---

### Layer 5
Synchronisation

Langfristige Abstimmung und Stabilisierung bestehender Informationsbeziehungen.

---

## Maschinenarchitektur

Jeder Layer kann zusätzlich in mehreren Formaten dargestellt werden:

- Markdown (.md)
- JSON (.json)
- YAML (.yaml)
- Schema / JSON-LD (.jsonld)

---

## Zweck der Maschinenarchitektur

Die maschinenlesbaren Dokumente dienen der strukturierten Beschreibung von Rollen, Informationsbeziehungen und logischen Zusammenhängen innerhalb der GEO-Methodik.

Sie stellen keine eigenständigen Inhalte dar, sondern eine technische Spiegelung der bestehenden Methodik.
